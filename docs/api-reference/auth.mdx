---
title: Auth API
---

Auth endpoints require a dashboard JWT session. They are used to manage API keys and retrieve the current user's profile.

## Get Current User

```http
GET /auth/me
Authorization: Bearer <jwt>
```

Returns the authenticated user and their organisation.

**Response** `200 OK`

```json
{
  "user": {
    "id": "u1b2c3d4-...",
    "org_id": "o1b2c3d4-...",
    "email": "admin@example.com",
    "role": "owner",
    "created_at": "2026-01-01T00:00:00Z"
  },
  "org": {
    "id": "o1b2c3d4-...",
    "name": "Acme Corp",
    "slug": "acme-corp",
    "created_at": "2026-01-01T00:00:00Z"
  }
}
```

**Errors**

| Status | Meaning |
|---|---|
| `401` | Missing or invalid JWT |

## Create API Key

```http
POST /api-keys
Authorization: Bearer <jwt>
```

**Request**

```json
{
  "name": "production-key"
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `name` | string | Yes | Human-readable label for the key |

**Response** `201 Created`

```json
{
  "id": "k1b2c3d4-...",
  "name": "production-key",
  "key": "ancra_live_xxxxxxxxxxxxxxxxxxxx",
  "created_at": "2026-01-01T10:00:00Z"
}
```

> Save the key immediately. The raw key is only returned once.

**Errors**

| Status | Meaning |
|---|---|
| `400` | Missing or invalid name |

## List API Keys

```http
GET /api-keys
Authorization: Bearer <jwt>
```

Returns all non-revoked keys. Key hashes are never returned.

**Response** `200 OK`

```json
{
  "keys": [
    {
      "id": "k1b2c3d4-...",
      "name": "production-key",
      "created_at": "2026-01-01T10:00:00Z",
      "last_used_at": "2026-01-02T10:00:00Z",
      "revoked_at": null
    }
  ]
}
```

---

## Revoke API Key

```http
DELETE /api-keys/{id}
Authorization: Bearer <jwt>
```

Revokes an API key immediately. Revoked keys are rejected by the auth middleware.

**Response** `200 OK`

```json
{ "status": "revoked" }
```

**Errors**

| Status | Meaning |
|---|---|
| `404` | Key not found |
