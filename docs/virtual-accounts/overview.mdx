# Virtual Accounts

A **Virtual Account** (DVA — Dedicated Virtual Account) is a real Nigerian bank account number assigned exclusively to one customer. When anyone transfers to that number via NIP (NIBSS Instant Payment), the funds are credited to the customer's ledger automatically.

## The Account Object

```json
{
  "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "customer_id": "c8a1b2d3-...",
  "account_ref": "c8a1b2d3-...",
  "bank_account_number": "9876543210",
  "bank_account_name": "Amara Okonkwo",
  "status": "active",
  "created_at": "2026-01-01T00:00:00Z"
}
```

| Field | Type | Description |
|---|---|---|
| `id` | UUID | Ancra's internal account ID |
| `customer_id` | UUID | The owning customer |
| `account_ref` | string | Nomba's idempotency key (= customer UUID) |
| `bank_account_number` | string | The 10-digit Nigerian bank account number |
| `bank_account_name` | string | The registered name on the account |
| `status` | string | `active` or `closed` |
| `created_at` | ISO 8601 | Provisioning timestamp |

## Idempotency

Each customer can have at most one active virtual account. Calling `POST /accounts` for a customer that already has an active account returns the existing account — no duplicate is created.

## Currency

All accounts are denominated in **NGN**. All balances are stored and returned in **kobo** (1 NGN = 100 kobo).
