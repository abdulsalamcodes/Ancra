# Balance & Transactions

## Get Balance

Returns the current ledger balance for an account.

```http
GET /accounts/{id}/balance
Authorization: Bearer <key>
```

```json
{
  "AccountID": "a1b2c3d4-...",
  "Balance": 250000,
  "Currency": "NGN",
  "AsOf": "2026-01-01T12:00:00Z"
}
```

`Balance` is in **kobo**. Divide by 100 to get NGN. `250000` kobo = ₦2,500.

---

## List Transactions

Returns a paginated list of ledger entries for an account, newest first.

```http
GET /accounts/{id}/transactions?limit=20&offset=0
Authorization: Bearer <key>
```

```json
{
  "Entries": [
    {
      "id": "e1b2c3d4-...",
      "account_id": "a1b2c3d4-...",
      "direction": "credit",
      "amount": 100000,
      "currency": "NGN",
      "entry_type": "inbound_credit",
      "external_ref": "API-VACT_TRA-...",
      "narration": "Transfer from John Doe",
      "created_at": "2026-01-01T11:00:00Z"
    }
  ],
  "Limit": 20,
  "Offset": 0
}
```

### Entry Fields

| Field | Type | Description |
|---|---|---|
| `id` | UUID | Ledger entry ID |
| `direction` | string | `credit` (money in) or `debit` (money out) |
| `amount` | integer | Amount in kobo |
| `currency` | string | Always `NGN` |
| `entry_type` | string | `inbound_credit`, `outbound_debit`, `transfer_reversal` |
| `external_ref` | string | Nomba transaction ID |
| `narration` | string | Payment narration from the sender |

## Account Statement

Returns entries with a **running balance** per entry, accurate across all pages. Useful for generating downloadable statements.

```http
GET /accounts/{id}/statement?limit=20&offset=0
Authorization: Bearer <key>
```

```json
{
  "account_id": "a1b2c3d4-...",
  "opening_balance": 0,
  "closing_balance": 250000,
  "currency": "NGN",
  "entries": [
    {
      "id": "e1b2c3d4-...",
      "direction": "credit",
      "amount": 250000,
      "running_balance": 250000,
      "entry_type": "inbound_credit",
      "narration": "Transfer from John Doe",
      "created_at": "2026-01-01T11:00:00Z"
    }
  ],
  "limit": 20,
  "offset": 0
}
```

`running_balance` is the account balance **after** each entry, correct even on paginated pages.
