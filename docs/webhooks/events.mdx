## payment_success

Fired when a NIP transfer successfully credits a virtual account.

```json
{
  "event_type": "payment_success",
  "requestId": "49e11b44-909b-4f83-82b4-9a83aXXXXXX",
  "data": {
    "merchant": {
      "walletId": "693e907aad9ea59616XXXX",
      "walletBalance": 539.4,
      "userId": "613bb620-c8e5-45f6-9c00-XXXXXXXX"
    },
    "terminal": {},
    "transaction": {
      "aliasAccountNumber": "9876543210",
      "fee": 0.6,
      "sessionId": "1000042602061021531516XXXXXX",
      "type": "vact_transfer",
      "transactionId": "API-VACT_TRA-613BB-eeae578a-...",
      "aliasAccountName": "Amara Okonkwo",
      "responseCode": "",
      "originatingFrom": "api",
      "transactionAmount": 1000,
      "narration": "Transfer from John Doe",
      "time": "2026-01-01T10:00:00Z",
      "aliasAccountReference": "c8a1b2d3-...",
      "aliasAccountType": "VIRTUAL"
    },
    "customer": {
      "bankCode": "305",
      "senderName": "John Doe",
      "bankName": "Paycom (Opay)",
      "accountNumber": "0812345678"
    }
  }
}
```

### Key Fields

| Field | Description |
|---|---|
| `data.transaction.aliasAccountNumber` | The virtual account that received the payment |
| `data.transaction.transactionAmount` | Amount in **NGN** (not kobo) |
| `data.transaction.transactionId` | Nomba's unique transaction ID — use for deduplication |
| `data.transaction.narration` | Sender's payment reference |
| `data.customer.senderName` | Name of the person who sent the money |

## payout_success

Fired when an outbound transfer completes successfully.

## payment_failed

Fired when an attempted payment fails before crediting. No ledger entry is created.

## payment_reversal

Fired when a previously credited payment is reversed. Contact support if you receive this event — manual ledger adjustment may be required.

## Unhandled Events

Ancra responds `200 {"status":"ignored"}` for any event type it does not handle. This prevents Nomba from retrying unrecognised events indefinitely.
