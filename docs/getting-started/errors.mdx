# Errors

Ancra uses standard HTTP status codes. All error responses share the same envelope.

## Error Envelope

```json
{
  "error": {
    "message": "human-readable description"
  }
}
```

## Status Codes

| Code | Meaning |
|---|---|
| `200` | Success |
| `201` | Resource created |
| `400` | Bad request — invalid input, missing fields, or malformed JSON |
| `401` | Unauthorized — missing or invalid API key / webhook signature |
| `404` | Not found |
| `422` | Unprocessable — semantically invalid request (e.g. insufficient funds) |
| `502` | Bad gateway — upstream payment provider error |
| `500` | Internal server error |

## Common Errors

### Missing or invalid API key

```json
{ "error": { "message": "unauthorized" } }
```

Make sure the `Authorization: Bearer <key>` header is present and the key has not been revoked.

### Insufficient funds (transfer)

```http
HTTP 422
```
```json
{ "error": { "message": "insufficient funds" } }
```

The source account's ledger balance is less than the requested transfer amount.

### Invalid UUID

```json
{ "error": { "message": "id must be a valid UUID" } }
```

Account and customer IDs are UUID v4. Verify the format before retrying.

### Transfer rejected by provider

```http
HTTP 502
```
```json
{ "error": { "message": "transfer rejected by payment provider" } }
```

Nomba declined the transfer. The ledger debit has been automatically reversed. Common causes: invalid destination account, account frozen, or daily limits exceeded.
